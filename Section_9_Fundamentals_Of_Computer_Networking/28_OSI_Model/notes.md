## OSI Model (Open Systems Interconnection Model)

    The OSI Model (Open Systems Interconnection Model) is a conceptual framework that defines how data
    travels through a network in 7 layers. Each layer has a specific role in moving data from one device to another.

## üèõ Layers of the OSI Model (Top ‚Üí Bottom)

### 7. Application Layer

- **Role:** The layer closest to the end-user.
- **Function:** Handles human-readable data (e.g., form inputs, POST requests).
- **Example:** A Browser sending a username and password to a Node.js server.
- **Protocols:** HTTP, HTTPS, FTP, SMTP.

### 6. Presentation Layer

- **Role:** Transforms data into a readable and secure format.
- **Tasks:**
- **Serialization:** (e.g., `JSON.stringify`).
- **Compression:** Reducing data size.
- **Encryption/Decryption:** Ensuring security.

- **Example:** HTTPS encryption, data compression.

### 5. Session Layer

- **Role:** Manages connections (sessions) between two devices.
- **Function:** Creates, maintains, and destroys sessions.
- **Example:** TCP connections, remote login sessions.
- **Note:** No changes are made to the actual data at this layer.

### 4. Transport Layer

- **Role:** Breaks data into smaller **Segments**.
- **Function:** Assigns Source and Destination **Port Numbers**.
- **Reliability:** Ensures reliable delivery (TCP) or fast delivery (UDP).
- **Delivery Type:** Service-to-Service delivery.
- **Example:** Segmenting data from Source Port 54321 to Destination Port 4000.

### 3. Network Layer

- **Role:** Adds IP addresses to segments to create **Packets**.
- **Function:** Assigns Source and Destination IP addresses.
- **Delivery Type:** End-to-End delivery.
- **Example:** Routing data from Source IP `192.168.0.5` to Destination IP `142.3.4.1`.

### 2. Data Link Layer

- **Role:** Adds MAC addresses to packets to create **Frames**.
- **Function:** Assigns Source and Destination MAC addresses (uses Router MACs during the hop, not just the final device).
- **Delivery Type:** Hop-to-Hop delivery.
- **Example:** Moving data between hardware via MAC address `24-9A-43-2B`.

### 1. Physical Layer

- **Role:** Converts frames into physical signals (electrical, optical, or radio waves).
- **Medium:** Data cables, Wi-Fi signals, fiber optics.
- **Note:** This is where data physically leaves the device.

---

## üì¶ Encapsulation & Decapsulation

- **Encapsulation:** As data moves from the **Application layer ‚Üí Physical layer**, each layer adds its specific header/information.
- **Decapsulation:** When data reaches the destination, each layer removes its header to pass the original data back up to the **Application layer**.

---

## üîë Delivery Types Summary

| Layer              | Delivery Type      | Address Used  |
| ------------------ | ------------------ | ------------- |
| **Transport (L4)** | Service-to-Service | Port Numbers  |
| **Network (L3)**   | End-to-End         | IP Addresses  |
| **Data Link (L2)** | Hop-to-Hop         | MAC Addresses |

---

## üìå Important Notes

- **Node.js Apps:** These reside in the **Application Layer**, but they indirectly utilize Presentation, Session, and Transport layer concepts.
- **Theory vs. Practice:** The OSI model is a **theoretical** framework. In practical networking, we mostly use the **TCP/IP Model**.
